Advanced Edge SLAM

The core of this system leverages the dual-core ESP32 microcontroller to handle real-time sensor fusion and mapping. By integrating Time-of-Flight (ToF) distance sensors with high-resolution encoders, the robot builds a local spatial model (SLAM) while serving an interactive UI over WebSocket protocols. This architecture minimizes latency and maximizes autonomy in constrained environments.

Operational Guidance

power Power & Grounding

expand_more
Ensure common ground between ESP32 and motor driver. High-current spikes from motors can induce I2C noise. Use 10uF decoupling capacitors near the ESP32 power pins for stability.

cable Wiring Standards

expand_more
Maintain short I2C cable runs (< 20cm). Use twisted pair for encoder signals to reduce EMI. All logic level shifting should be handled via dedicated bidirectional level shifters.

biotech Component Testing

expand_more
Run the `i2c_scanner` from the componentTesting directory first. If all sensors aren't detected, check pull-up resistors (4.7kΩ recommended for long buses).

healing I2C Recovery

expand_more
If the bus hangs, use the software-based I2C reset routine included in the firmware. This clears the SCL line by toggling it 9 times to release any slave devices stuck in a data cycle.
